Learning how to write queries takes time and practice, and writing valid queries is a critical skill in working with databases. Below are some suggestions about working with queries, including a syntax error checklist of common problems.

Complicated Queries:

  • Click the document below for guidance on writing complicated queries.

Running Time Issues:

  • If you've written your complex query and it is not running quickly, it is likely due to the query itself, rather than something wrong with the MySQL server.

  • Try running the complex query again, and wait at least a couple minutes for an answer. If the query is still running after 5 minutes or so, you have likely tried to implement a Cartesian product in your query, and should check the syntax. There is a syntax error checklist to help trouble shoot problems with queries.

Connection Issues:

  • If even the simple query won't execute, learners should try saving and closing the notebook.

  • To shutdown notebooks, click the "Running" tab on your Jupyter home page and click "Shutdown" on the right.

  • Restart the notebook. Usually these errors are temporary and will resolve themselves when our automated server monitoring notices an issue with the server and restarts the server. Open it, reconnect, and try the simple query again.

  • If the simple query still doesn't run, it likely means: (1) that there is a bad internet connection, or (2) that the partition has crashed due to bad queries (there are about 144 students per partition).

  • If we are made aware of a crashed partition, we will re-start it manually. Otherwise, all partitions are restarted once every 24 hours, so the system will be running again within a day (unless people crash it again with too many intensive queries).
